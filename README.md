# ChatNad 🤖

A lightweight, from-scratch ML chatbot  no deep learning, no external LLM APIs.
Built with **TF-IDF + SVM** intent classification, served with **Flask**, and
deployable to any standard Python host.

This project is intentionally built with **classical ML** (not neural networks),
so it's a good portfolio piece if you're comfortable with scikit-learn but haven't
done deep learning yet.

---

## How it works

1. `data/intents.json` holds example phrases ("patterns") grouped under intents
   like `greeting`, `joke`, `thanks`, etc.
2. `train.py` cleans and lemmatizes the text, converts it to TF-IDF vectors,
   and trains an SVM classifier to predict the intent behind any message.
3. `chatbot.py` loads the trained model. When a user sends a message, it
   predicts the intent and returns a random pre-written response for that
   intent — with a confidence threshold so low-confidence guesses fall back
   to "I don't understand" instead of a wrong answer.
4. `app.py` wraps `chatbot.py` in a Flask API + serves the chat web page.

```
User message → clean_text() → TF-IDF vector → SVM predicts intent (+ confidence)
   → if confidence too low: fallback response
   → else: random response from that intent's response list
```

## A → Z: Running it yourself

### 1. Set up the environment
```bash
python -m venv venv
source venv/bin/activate       # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Train the model
```bash
python train.py
```
This prints cross-validation accuracy and saves the trained model into `model/`.
Re-run this any time you edit `data/intents.json`.

### 3. Test it in the terminal (optional, quick sanity check)
```bash
python chatbot.py
```
Type messages, `quit` to exit.

### 4. Run the web app locally
```bash
python app.py
```
Open **http://127.0.0.1:5000** in your browser.

## Teaching ChatNad new things

Open `data/intents.json` and add a new block:
```json
{
  "tag": "your_new_topic",
  "patterns": ["example phrase 1", "example phrase 2", "..."],
  "responses": ["Response A", "Response B"]
}
```
Add **8–15 varied patterns** per intent for decent accuracy — more examples
= better generalization. Then re-run `python train.py`.

## Deployment

### Option A: Render / Railway (easiest, free tier available)
1. Push this folder to a GitHub repo.
2. On Render.com or Railway.app: "New Web Service" → connect your repo.
3. Build command: `pip install -r requirements.txt`
4. Start command: leave blank (Render/Railway will use the `Procfile`) —
   or explicitly set it to:
   ```
   python -c "import nltk; nltk.download('punkt'); nltk.download('punkt_tab'); nltk.download('wordnet'); nltk.download('omw-1.4')" && gunicorn app:app
   ```
5. Deploy. You'll get a public URL like `https://chatnad.onrender.com`.

### Option B: Docker (works anywhere — AWS, GCP, DigitalOcean, your own server)
```bash
docker build -t chatnad .
docker run -p 5000:5000 chatnad
```
Visit `http://localhost:5000`.

### Option C: Any VPS with Python
```bash
pip install -r requirements.txt
python -c "import nltk; nltk.download('punkt'); nltk.download('punkt_tab'); nltk.download('wordnet'); nltk.download('omw-1.4')"
gunicorn --bind 0.0.0.0:5000 app:app
```
Put nginx in front of it for a real domain + HTTPS (or use Render/Railway
which handle that for you).

## Making it a stronger resume project

- **Retrain and report metrics honestly.** The included cross-validation
  output in `train.py` gives you real numbers to talk about in interviews —
  don't inflate them.
- **Add more intents** relevant to a specific use case (e.g. a college FAQ
  bot, a product support bot) — recruiters respond better to a bot with a
  clear purpose than a generic one.
- **Add logging** of unanswered questions (low-confidence fallbacks) so you
  can see what users actually ask and expand `intents.json` accordingly —
  this is exactly how real production chatbots improve over time.
- **Write up your architecture decisions** in the repo's README (why TF-IDF
  + SVM instead of deep learning, how you chose the confidence threshold) —
  this shows engineering judgment, not just code output.
- **Next step toward deep learning:** once comfortable, swap the TF-IDF+SVM
  step for a small feedforward neural network (Keras `Dense` layers) trained
  on the same bag-of-words features — a natural, well-scoped intro to DL
  that builds directly on this project.

## Notes

- This is a **rule-based intent classifier**, not a generative model — it
  picks from pre-written responses rather than generating new text. That's
  a deliberate, honest scope for a first ML deployment project.
- The confidence threshold (`CONFIDENCE_THRESHOLD` in `chatbot.py`) controls
  how "cautious" ChatNad is about admitting it doesn't know something. Lower
  it for a more talkative bot, raise it if it's guessing wrong too often.
